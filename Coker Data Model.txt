Coker Data Model 

Space / Group 
-----
- dimension(coordinate system) + constraints
- Needs to be differentiable / tangent manifold.
fa- eg:
  - Reals,
  - Complex,
  - N-Vector
  - Tensor
  - SE3


DynamicQuantity (DQ)
---------------
- clock domain
- type (Cts/Discrete)


Signal
------
- name
- DQ


Variable
--------
- name
- parent (block)
- DQ


Parameter
---------
- name 
- parent (block)
- Space


Clock
-----
- Name
- Source (Global, Derived, Events) 


Model
-----
- Clock Domain
- Input Signal U
- Outputs Signal Y
- State X = [X, Z]

- Dynamics f(t, x, u, p) = D_t[x]
- Constraints h(t, x, u, p) = 0
- Output Map g(t, x, u , p) = y

- Component & Generating Parameters


Components
----------
- Function that takes parameters and builds Models
- Associated GUI Plugin
- Components are defined using Python or C++

Workflows
---------
- Workbench (Build, simulate, Interact)
- Lab (Analyse & Optimise) 
- Factory (Compile & Deploy)



* Workflows are just a python file.
  Workflows have a "default component", which is a `fn(p) -> Model`
  with:
	- No Inputs
	'Actions' are just adding a line to the fn
	'Undo' removes a line
	'Redo' re-adds a line
 Workflows have a "main" function
